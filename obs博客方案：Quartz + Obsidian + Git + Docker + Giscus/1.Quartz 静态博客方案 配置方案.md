好的，我们把 **Quartz 静态博客方案** 完整地梳理一遍，涵盖从零开始部署、选择性发布、评论集成，到自动化发布的全流程。这是一份你可以直接照着操作的指南。

## 0.1 📦 方案概览：Quartz + Obsidian + Git + Docker + Giscus

| 组件 | 作用 |
| --- | --- |
| **Quartz** | 静态博客生成器，原生支持 Obsidian 双向链接、图谱等特性 |
| **Obsidian** | 写作工具，通过插件与 Git 联动 |
| **Git + GitHub** | 存储笔记源码，触发自动部署 |
| **Docker** | 在腾讯云上运行 Quartz 构建和服务 |
| **Giscus** | 基于 GitHub Discussions 的评论系统，与 Quartz 无缝集成 |

## 0.2 🖥️ 第一步：服务器端部署（腾讯云轻量服务器）

### 0.2.1 1\. 安装 Docker 和 Docker Compose

bash

sudo apt update
sudo apt install apt-transport-https ca-certificates curl software-properties-common
curl \-fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg \--dearmor \-o /usr/share/keyrings/docker-archive-keyring.gpg
echo "deb \[arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg\] https://download.docker.com/linux/ubuntu $(lsb_release \-cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list \> /dev/null
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo usermod \-aG docker $USER
\# 重新登录使组生效
exit

### 0.2.2 2\. 克隆 Quartz 并创建 Docker 配置

bash

mkdir \-p ~/apps/my-digital-garden
cd ~/apps/my-digital-garden
git clone https://github.com/jackyzha0/quartz.git quartz-source
cd quartz-source

创建 `docker-compose.yml`：

yaml

version: '3.8'
services:
  quartz:
    build: .
    container_name: quartz\-garden
    ports:
      \- "8080:8080"
    volumes:
      \# 稍后会用软链接将 content 指向你的 Obsidian 仓库
      \- /path/to/your/obsidian/vault:/usr/src/app/content
    restart: unless\-stopped
    environment:
      \- NODE_ENV=production

### 0.2.3 3\. 用软链接连接你的 Obsidian 仓库

假设你的 Obsidian 仓库在 `/home/ubuntu/my-obsidian-vault`（根据实际情况修改）：

bash

cd ~/apps/my-digital-garden/quartz-source
rm \-rf content
ln \-s /home/ubuntu/my-obsidian-vault content

### 0.2.4 4\. 首次启动并验证

bash

docker compose up \-d

访问 `http://你的服务器IP:8080` 即可看到默认页面（需在腾讯云防火墙开放 8080 端口）。

## 0.3 🔌 第二步：配置选择性发布（只公开部分笔记）

Quartz 提供了两种方式控制哪些笔记被发布：

### 0.3.1 方式一：通过 `ignorePatterns` 忽略文件夹（推荐）

编辑 `quartz.config.ts`，在 `configuration` 中添加 `ignorePatterns`：

typescript

const config: QuartzConfig \= {
  configuration: {
    // ... 其他配置
    ignorePatterns: \["private", "templates", "daily", ".obsidian"\],
  },
  // ...
}

这样，`private/`、`templates/` 等文件夹下的笔记将完全不会被构建。

### 0.3.2 方式二：通过 Frontmatter 标记（更精细的控制）

在笔记的 frontmatter 中添加自定义字段，例如 `publish: true`，然后结合 `Plugin.FilterContent` 插件（需要修改 `quartz.config.ts` 中的插件列表）来只发布标记为 `true` 的笔记。这种方法灵活性高，但需要一点配置。

**配置步骤**：

1. 编辑 `quartz.config.ts`，找到 `plugins` 部分。
2. 在 `filters` 数组中添加 `Plugin.FilterContent()`，并指定检查的 frontmatter 字段：

typescript

plugins: {
  filters: \[
    Plugin.FilterContent({ markdownOnly: true }), // 只处理 Markdown 文件
    Plugin.FilterContent({ frontmatter: { publish: true } }), // 只发布 frontmatter 中有 publish: true 的笔记
  \],
  // ... 其他插件
}

3. 这样，只有 frontmatter 包含 `publish: true` 的笔记才会被纳入博客。

> **注意**：`ignorePatterns` 和 `FilterContent` 可以结合使用，前者粗粒度排除文件夹，后者细粒度选择笔记。

## 0.4 💬 第三步：集成评论系统 Giscus

### 0.4.1 1\. 准备工作

- 确保你的博客源码仓库（GitHub）是 **公开** 的。
- 在仓库中启用 **Discussions**（Settings → General → Features → Discussions）。
- 安装 [Giscus GitHub App](https://github.com/apps/giscus) 到该仓库。

### 0.4.2 2\. 获取 Giscus 配置参数

访问 [giscus.app](https://giscus.app/)，填写你的仓库名，选择 “Announcements” 作为 Discussion 分类，网站会生成 `repoId`、`categoryId` 等参数。

### 0.4.3 3\. 修改 Quartz 布局文件启用评论

编辑 `quartz.layout.ts`，在 `afterBody` 部分添加 `Component.Comments`：

typescript

import { Component } from "./quartz/components"
// ...
afterBody: \[
  Component.Comments({
    provider: 'giscus',
    options: {
      repo: '你的用户名/你的仓库名',
      repoId: '你获取的 repoId',
      category: 'Announcements',
      categoryId: '你获取的 categoryId',
      mapping: 'pathname', // 或 'title' 等
      reactionsEnabled: true,
      emitMetadata: true,
      // 其他选项
    }
  }),
\],

### 0.4.4 4\. （可选）为单篇文章控制评论显示

你可以在笔记 frontmatter 中添加 `comments: false` 来关闭某篇文章的评论。Giscus 组件会自动检查该字段。

## 0.5 🔄 第四步：自动化发布（Git + Webhook / GitHub Actions）

实现 Obsidian 写作后自动更新博客。

### 0.5.1 方案 A：GitHub Actions + SSH（推荐）

1. **在服务器上生成 SSH 密钥**（无需密码）：
	bash
	ssh-keygen \-t ed25519 \-f ~/.ssh/github-actions-key
2. **将公钥添加到 `~/.ssh/authorized_keys`**：
	bash
	cat ~/.ssh/github-actions-key.pub \>> ~/.ssh/authorized_keys
3. **在 GitHub 仓库中添加 Secrets**：
	- `SERVER_SSH_PRIVATE_KEY`：私钥文件内容
	- `SERVER_HOST`：服务器 IP
	- `SERVER_USERNAME`：登录用户名
4. **在仓库中创建 GitHub Actions 工作流**：`.github/workflows/deploy.yml`
	yaml
	name: Deploy to Server
	on:
	  push:
	    branches: \[ main \]  \# 或你使用的分支
	jobs:
	  deploy:
	    runs-on: ubuntu\-latest
	    steps:
	      \- name: Checkout
	        uses: actions/checkout@v3
	      \- name: Deploy via SSH
	        uses: appleboy/ssh\-action@v0.1.5
	        with:
	          host: ${{ secrets.SERVER_HOST }}
	          username: ${{ secrets.SERVER_USERNAME }}
	          key: ${{ secrets.SERVER_SSH_PRIVATE_KEY }}
	          script: |
	            cd ~/apps/my-digital-garden/quartz-source
	            git pull
	            docker compose exec quartz npx quartz build

### 0.5.2 方案 B：服务器定时任务（cron）

如果不想用 GitHub Actions，可以在服务器上设置每 10 分钟拉取一次：

bash

crontab \-e
\# 添加一行
\*/10 \* \* \* \* cd /home/你的用户名/apps/my-digital-garden/quartz-source && git pull && docker compose exec quartz npx quartz build \> /dev/null 2\>&1

## 0.6 📝 第五步：日常写作与发布流程

1. **在 Obsidian 中写作**：像平常一样创建、编辑笔记。
2. **标记发布**：在笔记 frontmatter 中添加 `publish: true`（如果使用 FilterContent 方式）。
3. **提交推送**：使用 Git 插件（如 Obsidian Git）或手动提交，将更改推送到 GitHub 仓库的 `main` 分支。
4. **自动更新**：GitHub Actions 或服务器定时任务自动拉取并重建博客。
5. **查看效果**：访问你的博客域名，新文章上线，评论区也已就绪。

## 0.7 🔐 第六步：配置 HTTPS（强烈推荐）

使用 Nginx Proxy Manager 自动申请 Let's Encrypt 证书。

### 0.7.1 1\. 部署 Nginx Proxy Manager

bash

mkdir ~/apps/nginx-proxy-manager
cd ~/apps/nginx-proxy-manager
nano docker-compose.yml

yaml

version: '3'
services:
  aimage: 'jc21/nginx-proxy-manager:latest'
    restart: unless\-stopped
    ports:
      \- '80:80'
      \- '443:443'
      \- '81:81'
    volumes:
      \- ./data:/data
      \- ./letsencrypt:/etc/letsencrypt

bash

docker compose up \-d

访问 `http://你的服务器IP:81`，默认账号 `admin@example.com` / `changeme`，首次登录强制修改密码。

### 0.7.2 2\. 配置反向代理

- 添加 Proxy Host
- Domain Names：填写你的域名（如 `blog.yourdomain.com`）
- Forward Hostname / IP：`127.0.0.1` 或 `你的服务器内网IP`
- Forward Port：`8080`（Quartz 容器映射的端口）
- 打开 Cache Assets、Block Common Exploits
- 切换到 SSL 标签，选择 Request a new SSL Certificate，勾选 Force SSL

几分钟后即可通过 `https://你的域名` 安全访问。

## 0.8 🧹 总结

通过这套 Quartz 方案，你获得了：

- **沉浸式写作**：在 Obsidian 中自由创作，使用双向链接、图谱等功能。
- **选择性发布**：通过文件夹忽略或 frontmatter 标记，精准控制公开内容。
- **原生评论**：Giscus 集成，无需自建后端，评论数据托管在 GitHub Discussions。
- **自动化部署**：Git push 触发服务器重建，博客实时更新。
- **安全访问**：Nginx 反向代理 + 免费 HTTPS。

这套方案成熟稳定，且完全开源免费。现在你可以专注于写作，剩下的交给自动化流程。如果在配置中遇到任何问题，欢迎随时提问！
