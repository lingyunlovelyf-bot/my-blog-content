你误解了命令：`.github/workflows/deploy.yml` 是一个**文件路径**，不是可执行的命令。你需要在你的本地仓库中**创建这个文件**，然后通过 Git 提交推送。以下是正确的操作步骤：

---

### 📁 第一步：创建 `.github/workflows/deploy.yml` 文件

在 Windows 的 cmd 中，依次执行以下命令（确保当前目录是 `G:\Obsidian\02我的笔记`）：

bash

\# 创建 .github 文件夹和 workflows 子文件夹
mkdir .github
mkdir .github\\workflows
\# 用记事本创建并编辑 deploy.yml 文件
notepad .github\\workflows\\deploy.yml

如果 `notepad` 命令无法打开，也可以手动在资源管理器中进入 `G:\Obsidian\02我的笔记`，创建 `.github` 文件夹，再在里面创建 `workflows` 文件夹，然后右键新建一个文本文件，重命名为 `deploy.yml`（注意扩展名是 `.yml`，不是 `.txt`）。然后用记事本或 VS Code 打开编辑。

---

### ✏️ 第二步：写入正确的 YAML 内容

将以下内容**完整复制**到 `deploy.yml` 文件中（请根据你的实际情况微调路径，但按你之前的配置应该不用改）：

yaml

name: Deploy to Server
on:
  push:
    branches: \[ main \]
jobs:
  deploy:
    runs-on: ubuntu\-latest
    steps:
      \- name: Checkout code
        uses: actions/checkout@v3
      \- name: Deploy via SSH
        uses: appleboy/ssh\-action@v0.1.5
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USERNAME }}
          key: ${{ secrets.SERVER_SSH_PRIVATE_KEY }}
          script: |
            cd /root/my-blog-content
            git pull
            cd /root/apps/my-digital-garden/quartz-source
            docker compose exec quartz npx quartz build

**注意**：最后一行 `docker compose exec quartz npx quartz build` 中的 `quartz` 是你的 Docker 服务名。如果你在 `docker-compose.yml` 中定义的服务名不是 `quartz`（例如是 `app`），请修改为对应的服务名。你可以登录服务器，进入 `~/apps/my-digital-garden/quartz-source`，查看 `docker-compose.yml` 中 `services:` 下面的名称。

保存文件后关闭记事本。

---

### 📤 第三步：提交并推送文件

在 cmd 中继续执行：

bash

git add .github/workflows/deploy.yml
git commit \-m "Add GitHub Actions auto deploy workflow"
git push

推送成功后，前往 GitHub 仓库的 **Actions** 标签页，你应该能看到名为 "Deploy to Server" 的工作流正在运行或已完成。

---

### ✅ 第四步：测试自动更新

1. 在本地修改任意一篇笔记（例如 `index.md` 加一句话）。
2. 执行：
	bash
	git add .
	git commit \-m "Test auto deploy"
	git push
3. 等待约 1 分钟，刷新你的博客，查看是否更新。

---

### ❓ 如果 Actions 运行失败

请点击进入该工作流的运行详情，查看红色的错误步骤，并将错误日志复制给我。常见问题包括：

- SSH 认证失败（检查 Secrets 中的私钥是否正确，公钥是否已添加到 GitHub）。
- 服务器上路径不正确（例如 `cd /root/my-blog-content` 失败）。
- `docker compose` 命令不存在（服务器未安装 Docker Compose V2）。

我会根据日志帮你解决。
