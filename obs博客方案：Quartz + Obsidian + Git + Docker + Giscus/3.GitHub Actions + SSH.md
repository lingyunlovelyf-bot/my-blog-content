---

## 🔧 准备工作确认

- 你的服务器上已经运行着 Quartz 容器（容器名 `quartz-garden`）。
- 你的内容仓库在服务器上的路径为：`/root/my-blog-content`（通过 `git clone` 获得）。
- 你的本地笔记仓库已经与 GitHub 远程仓库 `https://github.com/lingyunlovelyf-bot/my-blog-content.git` 关联。
- 你已经在服务器上生成了 SSH 密钥对（例如 `~/.ssh/github-deploy-key`），并且**公钥已添加到 GitHub 账户或仓库的 Deploy Keys 中**（确保有读取权限）。
- 你已经在 GitHub 仓库的 **Settings → Secrets and variables → Actions** 中添加了以下三个 Secrets：
	- `SERVER_HOST`：你的服务器公网 IP
	- `SERVER_USERNAME`：你的服务器用户名（你一直用 `root`）
	- `SERVER_SSH_PRIVATE_KEY`：**私钥全文**（`cat ~/.ssh/github-deploy-key` 的内容，包括 `BEGIN` 和 `END` 行）

如果还有哪个 Secret 没添加，请先补上。

---

## 📝 第一步：创建 GitHub Actions 工作流文件

在你的本地仓库（`G:\Obsidian\02我的笔记`）中，创建以下目录和文件：

bash

.github/workflows/deploy.yml

用文本编辑器打开 `deploy.yml`，粘贴以下内容（**请根据你的实际路径微调**）：

yaml

name: Deploy to Server
on:
  push:
    branches: \[ main \]   \# 如果你用的分支不是 main，请修改
jobs:
  deploy:
    runs-on: ubuntu\-latest
    steps:
      \- name: Checkout code
        uses: actions/checkout@v3
      \- name: Deploy via SSH
        uses: appleboy/ssh\-action@v0.1.5
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USERNAME }}
          key: ${{ secrets.SERVER_SSH_PRIVATE_KEY }}
          script: |
            cd /root/my-blog-content
            git pull
            cd /root/apps/my-digital-garden/quartz-source
            docker compose exec quartz npx quartz build

### 🔍 关键点解释

- `cd /root/my-blog-content`：进入内容仓库，拉取最新笔记。
- `cd /root/apps/my-digital-garden/quartz-source`：进入 Quartz 项目目录。
- `docker compose exec quartz npx quartz build`：在运行中的容器内执行构建命令。注意这里的 `quartz` 是 `docker-compose.yml` 中定义的服务名，如果你在 `docker-compose.yml` 中定义的服务名不是 `quartz`（例如是 `app`），请相应修改。

保存文件。

---

## 📤 第二步：提交并推送工作流文件

在本地仓库目录下执行：

bash

git add .github/workflows/deploy.yml
git commit \-m "Add auto deploy workflow"
git push

推送成功后，前往 GitHub 仓库的 **Actions** 标签页，应该能看到一个名为 "Deploy to Server" 的工作流正在运行。点击它可以查看实时日志。

---

## ✅ 第三步：测试自动更新

1. 在本地修改一篇笔记（例如在 `index.md` 中添加一句话）。
2. 提交并推送：
	bash
	git add .
	git commit \-m "Test auto update"
	git push
3. 等待几分钟（通常 30 秒左右），刷新你的博客页面，应该能看到刚刚修改的内容。

## ❓ 常见问题排查

| 问题 | 可能原因 | 解决方法 |
| --- | --- | --- |
| Actions 日志显示 `Permission denied (publickey)` | 私钥未正确添加到 Secrets，或公钥未添加到 GitHub | 检查 Secrets 中的私钥是否完整；确认公钥已添加到 GitHub（账户 SSH keys 或仓库 Deploy keys）。 |
| `git pull` 失败 | 服务器上的仓库未设置远程分支，或仓库是私有的但未配置认证 | 登录服务器，进入 `/root/my-blog-content` 手动执行 `git pull` 看是否正常。如果仓库私有，确保 SSH 密钥已添加到 Deploy keys 且有读取权限。 |
| `docker compose` 命令未找到 | 服务器未安装 Docker Compose V2 | 登录服务器，运行 `docker compose version` 检查。如未安装，参考之前步骤安装。 |
| 构建成功但博客未更新 | 浏览器缓存；Quartz 过滤规则（如 `RemoveDrafts`）导致新笔记被忽略 | 强制刷新浏览器；检查笔记 frontmatter 是否包含 `draft: false`（如果使用了该插件）。 |

## 📌 后续维护

- 以后每次在 Obsidian 中写完笔记，只需提交并推送到 GitHub，Actions 会自动完成服务器拉取和构建。
- 如果修改了 Quartz 配置（如 `quartz.config.ts`），也需要提交并推送，但 Actions 中的脚本只会拉取内容仓库，不会拉取 Quartz 源码。如需更新 Quartz 源码，需手动登录服务器 `git pull` 对应目录。
