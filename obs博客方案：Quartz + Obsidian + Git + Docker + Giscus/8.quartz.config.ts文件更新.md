### 🔍 问题诊断

你的配置文件**存在重复和语法错误**，导致 Quartz 无法正确读取设置。

1. **致命的重复定义**：你在文件末尾**又定义了一次** `const config: QuartzConfig = {...}`，并且包含了新的 `ignorePatterns`。这在 TypeScript 中是不允许的，会覆盖前面的配置或直接报错。
2. **`ignorePatterns` 配置错误**：你希望忽略的文件夹（如 `01备忘录`）被写在了后面那个重复的 `config` 里。由于重复定义，这个规则可能根本没有生效。
3. **`RemoveDrafts` 已注释**：你注释掉了 `Plugin.RemoveDrafts()`，这是正确的，避免了笔记因缺少 `draft: false` 而被过滤。
4. **`baseUrl` 未修改**：`baseUrl` 仍是默认的 `quartz.jzhao.xyz`，你应该把它改成你自己的域名（例如 `garden.yourdomain.com`）。

### 🛠️ 解决方案：提供一个干净正确的配置文件

### 📝 修改后操作步骤

1. **登录服务器**，用 `nano` 或其他编辑器打开配置文件：
	bash
	nano /root/apps/my-digital-garden/quartz-source/quartz.config.ts
2. **删除所有现有内容**，粘贴上面提供的**完整正确代码**。
3. **修改关键字段**：
	- 将 `baseUrl` 的值改成你的实际域名（例如 `garden.yourdomain.com`）。如果还没域名，可以先删除这一行或留空字符串 `""`。
	- 可以根据喜好修改 `pageTitle`。
4. **保存退出**（`Ctrl+O` 回车，`Ctrl+X`）。
5. **重新构建博客**：
	bash
	cd /root/apps/my-digital-garden/quartz-source
	docker compose exec quartz npx quartz build
6. **查看构建输出**，确认 `Found X input files` 的数量是你内容仓库中的笔记总数，且 `Filtered out` 应该是 0。
7. **验证访问**：直接访问 `http://你的服务器IP:8080`，现在应该能看到正常的博客首页了。
